# USIM-AGENT: Техническое задание на разработку v0.2

**Версия:** 0.2  
**Статус:** extended scope  
**Контекст:** развитие v0.1 в сторону более робастной, интерпретируемой и управляемой системы.

---

## 1. Цель версии

Версия v0.2 расширяет baseline v0.1 четырьмя направлениями:
1. Робастность LLM-компиляции.
2. Управление вычислительной сложностью.
3. Явная обработка неопределённости.
4. Интерфейс для ручной корректировки модели человеком.

---

## 2. Наследуемая база v0.1

В v0.2 сохраняются все базовые компоненты v0.1:
- 6-модульная архитектура;
- session mode;
- WorldGraph + Function Registry;
- Verifier уровней 0, 1, 2;
- fixed-point iteration;
- Knowledge Graph + Session Reconciler;
- query/status/export interface.

---

## 3. Новые требования v0.2

### 3.1 Робастность LLM-компиляции

После `compile_domain()` добавляется слой `compile_validation()`.

#### Требования:
- Backtesting compiled domain на исторических данных.
- Сравнение траекторий модели с историческими наблюдениями.
- Повторная компиляция при провале validation.
- Ensemble compile: 2–3 независимых варианта компиляции.
- Sign voting / consensus по causal edges.
- Проверка конфликтов между новым causal DAG и существующим KG.
- Конфликтный domain compile помечается как `review_required`.

#### Новые сущности:
- `CompileCandidate`
- `CompileValidationReport`
- `HistoricalFitScore`

#### Новые параметры конфига:
- `compile_retries`
- `compile_ensemble_size`
- `backtest_horizon`
- `historical_fit_threshold`
- `sign_conflict_action: warn | block | review`

---

### 3.2 Управление вычислительной сложностью

`cost_i` должен стать формальной функцией, а не абстрактной величиной.

#### Требования:
- Явная cost-модель задачи:
  - число LLM-вызовов;
  - число шагов симуляции;
  - число акторов / ресурсов;
  - число доменов в MultiDomainWorld;
  - объём обновлений KG.
- Оценка expected_cost до запуска задачи.
- Учёт remaining attention/computation budget.
- Ограничение глубины `DEEP_DIVE`.
- Принудительная остановка или downgrade при превышении budget.

#### Новые ограничения:
- `max_llm_calls_per_task`
- `max_sim_steps_per_task`
- `max_domains_per_task`
- `max_deep_dive_depth`
- `max_compute_budget_per_session`

#### Логирование:
- `estimated_cost`
- `actual_cost`
- `stop_reason`
- `budget_remaining`

---

### 3.3 Обработка неопределённости

v0.2 вводит явное вероятностное представление параметров и исходов.

#### Требования:
- Параметры домена допускают распределения, а не только point estimates.
- Поддержка Monte Carlo / scenario sampling.
- Outcome scoring возвращает распределение исходов.
- Confidence выводится из устойчивости распределения, а не только из point score.
- Export результатов должен включать интервалы неопределённости.

#### Новые сущности:
- `ParameterDistribution`
- `ScenarioSample`
- `OutcomeDistribution`
- `ConfidenceReport`

#### Новые параметры конфига:
- `monte_carlo_samples`
- `parameter_prior_type`
- `uncertainty_threshold`
- `confidence_from_variance: true`

---

### 3.4 Human-in-the-loop editing

Human Interface становится не только read-only, но и editable.

#### Требования:
- Человек может менять параметры симуляции.
- Человек может менять знак causal edge.
- Человек может подтверждать / отклонять compiled domain.
- После правки возможен повторный прогон симуляции.
- Интерфейс показывает diff результатов до/после override.
- Любая ручная правка сохраняется в audit log.
- Правки не удаляют исходную машинную гипотезу, а создают новую версию.

#### Новые API:
- `PATCH /domain/{id}/params`
- `PATCH /domain/{id}/edges/{edge_id}`
- `POST /domain/{id}/rerun`
- `GET /domain/{id}/diff`

#### Новые CLI:
- `override-param`
- `override-sign`
- `rerun-domain`
- `diff-domain`

## 4. Изменения по модулям

### Module 2: Attention Scheduler
- Добавляется cost governance.
- UCB учитывает формальную cost-функцию.
- DEEP_DIVE запускается только после budget pre-check.

### Module 3: USim Core + Verifier
- Добавляется probabilistic scoring.
- Compile validation становится обязательным этапом перед основным simulation run.

### Module 5: Knowledge Graph
- Добавляются human override events как отдельный тип evidence.
- Поддерживается версия знания: machine hypothesis / human-adjusted hypothesis.

### Module 6: Human Interface
- Добавляются формы/endpoint’ы редактирования.
- Добавляется diff-view симуляций и подтверждение человеком.

---

## 5. Что входит в v0.2

### Входит
- Validation layer для LLM compile.
- Cost model и compute budget policy.
- Probabilistic uncertainty layer.
- Human override interface.
- Расширенные session logs.
- Расширение export результатами uncertainty и override history.

### Не входит
- Full fine-tuning LLM.
- Federated multi-agent memory.
- Полностью автоматическое открытие новых доменов без LLM.
- Production-grade distributed execution.

---

## 6. Нефункциональные требования v0.2

- Все compile decisions должны быть воспроизводимы.
- Все human overrides должны быть аудируемы.
- Все probabilistic runs должны иметь контролируемый computational budget.
- Любая правка модели человеком должна быть обратима.
- История версий модели должна сохраняться.

## 9. Критерии готовности v0.2

Версия считается готовой, если:
- compile validation отбрасывает неадекватные domain compile;
- historical fit оценивается автоматически;
- cost governance предотвращает runaway analysis;
- probabilistic scoring возвращает устойчивые outcome distributions;
- human overrides корректно влияют на rerun;
- diff результатов до/после правки доступен пользователю;
- audit trail полон и воспроизводим.

---

## 10. Результат версии

Результат v0.2 — более надёжный и интерпретируемый аналитический агент, который:
- лучше контролирует ошибки LLM-компиляции;
- ограничивает вычислительную сложность;
- явно представляет неопределённость;
- допускает содержательную корректировку модели человеком.
