# USIM-AGENT: Техническое задание на разработку v0.1

**Версия:** 0.1  
**Статус:** baseline scope  
**Контекст:** автономный аналитический агент на базе универсального симулятора (USim) и LLM.

---

## 1. Цель версии

Версия v0.1 должна реализовать минимально работоспособного автономного агента, который:
- принимает сигналы из внешних источников;
- выбирает задачи через механизм распределения внимания;
- компилирует домен в симуляционную модель;
- прогоняет симуляцию с автоматической верификацией;
- сохраняет результаты в граф знаний;
- позволяет человеку запрашивать статус, задавать вопросы и экспортировать граф знаний.

---

## 2. Архитектура

Система состоит из шести модулей:

1. Signal Ingestion Layer
2. Attention Scheduler
3. USim Core + Verifier
4. Domain Registry
5. Knowledge Graph (LTM)
6. Human Interface

Агент работает в сессионном режиме:
- сессия 2–4 часа;
- затем reconciliation;
- затем ожидание следующей сессии.

---

## 3. Функциональный scope v0.1

### 3.1 Signal Ingestion
- Подключение источников: Tavily, RSS, structured API, manual input.
- Первичная нормализация текста в сигналы: topic, entities, sentiment, timestamp.
- Статистический trigger через Mahalanobis distance.
- Семантический trigger через LLM-классификацию shock type.
- Режимы: WATCH, ANALYZE, DEEP_DIVE.

### 3.2 Attention Scheduler
- Инвариант конечного attention budget.
- Выбор задач по формуле expected information gain / cost.
- UCB-аллокация между competing tasks.
- Машина состояний задач:
  `PENDING → ACTIVE → SUSPENDED → COMPLETED → ARCHIVED`.

### 3.3 USim Core
- Абстрактный WorldGraph:
  actors, resources, relations, outcome_registry.
- Function Registry:
  linear, stock_flow, logistic, threshold, coupled.
- Общий оператор перехода:
  `S_{t+1} = S_t + F_{θ_D}(S_t, π_t)`.
- Outcome scoring:
  `p(o_t) = softmax(W · S_t)`.

### 3.4 Verifier
В v0.1 реализуются:
- Level 0: математические инварианты;
- Level 1: структурные тесты;
- Level 2: sign-consistency по causal DAG.

### 3.5 Fixed Point
- Итерационная коррекция параметров при циклических зависимостях.
- Проверка spectral radius как условия сходимости.

### 3.6 Knowledge Graph
- Хранение узлов и рёбер знаний с confidence.
- Статусы: active, uncertain, review, archived.
- Query по KG.
- Export: HTML / JSON / DOT.

### 3.7 Session Reconciler
- Извлечение knowledge deltas из session log.
- Обнаружение конфликтов.
- Beta update confidence.
- Архивация слабых узлов.
- Обновление Domain Registry.

### 3.8 Human Interface
В v0.1 интерфейс read-only:
- query interrupt;
- status endpoint;
- knowledge export.

## 4. Что входит в v0.1

### Входит
- Один агент.
- Один pipeline compile → simulate → verify → score → update KG.
- Один домен за запуск analysis pipeline.
- DEEP_DIVE через повторный запуск с увеличением горизонта и/или доменов.
- NetworkX + JSON persistence как backend памяти.
- CLI и API для запуска, статуса, query, reconcile, export.

### Не входит
- Fine-tuning LLM.
- Мультиагентный режим.
- Federated KG.
- Полноценное ручное редактирование модели человеком.
- Явная вероятностная симуляция.
- Полноценная cost-governance модель.

---

## 5. Нефункциональные требования

- Все решения и нарушения verifier логируются.
- Нарушение Level 0 останавливает симуляцию.
- Query interrupt не должен терять состояние активной сессии.
- Все знания должны быть аудируемы через KG и session logs.
- Конфигурация хранится в YAML.

## 8. Критерии готовности v0.1

Версия считается готовой, если:
- проходят unit tests ядра;
- проходят tests verifier уровней 0–2;
- attention scheduler соблюдает budget invariant;
- query interrupt работает без потери состояния;
- агент проходит интеграционный тест сессии 30 минут;
- reconciliation и export KG выполняются без ошибок.
